## 直接操作寄存器
需要一步一步找数据手册，知道外设
### RCC->APB2ENR//开启GPIOC时钟

![[Pasted image 20260307110217.png]]
- 由观察STM32F103C8T6可知，灯接在PC13引脚上，所以这边选择位4 IOPCRST，置1开启GPIOC模块的时钟信号
- 该步骤为第一步，如果GPIOC为0，后续操作都不会开启，因为模块没通电进行工作
- 由此可知，除了位4为1之外其他都为0，在16进制中，为00000010
### GPIOC->CRH//配置PC13为推挽输出模式
![[Pasted image 20260307110852.png]]
- 因为是控制PC13口，每个端口由4位进行配置，那16个端口需要64位，13是高位寄存器，这边选择控制20-23
- 22-23CNF13=00(推挽输出模式)
- 推挽输出模式：点亮LED需要强力的高低电平才能发光或熄灭，推挽输出模式由两个MOS管组合成，一个“推”(输出高电平，连接VDD)，一个“挽”(输出低电平，连接VSS)，可以更稳定的控制LED
- 20-21MODE13=11(输出模式，50MHz)
- 50Hz是STM32F103的最高规格，在任何场景下都不会因为速度不够而出问题
- 在现在的LED换其他的频率没问题，如果是其他情况下配置不合适的频率，会导致波形失真
### GPIOC->ODR//控制亮灭
![[Pasted image 20260307112611.png]]

由于PC13是低电平点亮，所以0X00000000是亮，0X00002000是灭
### 程序
```c
#include "stm32f10x.h"                  // Device header
int main(void)
{
	RCC->APB2ENR = 0x00000010;//开启GPIOC时钟
	GPIOC->CRH = 0X00300000;//配置PC13为推挽输出模式
	//GPIOC->ODR = 0X00002000;//PC13灭
	GPIOC->ODR = 0X00000000;//PC13亮（低电平点亮）
	while(1)
	{
	}
}
```
## 用标准库函数
先新建一个“Library”文件夹，把官方的标准库函数放进去
原理上类似，但是使用了官方的标准库来进行编程，官方在函数中写好了用就行
本质上还是操作寄存器，寄存器跟上面一样，所以函数的顺序


```c
#include "stm32f10x.h"                  // Device header
int main(void)
{
	RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOC,ENABLE);
	//定义结构体
	GPIO_InitTypeDef GPIO_InitStructure;
	GPIO_InitStructure.GPIO_Mode =GPIO_Mode_Out_PP;
	GPIO_InitStructure.GPIO_Pin =GPIO_Pin_13;
	GPIO_InitStructure.GPIO_Speed =GPIO_Speed_50MHz;
	
	GPIO_Init(GPIOC,&GPIO_InitStructure);//在说明中说需要定义结构体，然后这是一个指向结构体的指针
	
	GPIO_SetBits(GPIOC,GPIO_Pin_13);//高电平灭
	//GPIO_ResetBits(GPIOC,GPIO_Pin_13);//低电平亮
		while(1)
	{
	}
}

```
