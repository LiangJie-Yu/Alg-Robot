## 定义
把一个大程序拆成很多.c(源文件)和.h(头文件)文件
## 目的
1. 代码清晰
2. 便于复用
3. 方便很多人写
4. 方便移植
## `#include`规则
- `include`后面只能跟`.h`
- 绝对不能`#include`任何`.c`文件
- 包含`.h`=声明声明内容进来
- 包含`.c`=复制实现代码，必定重复定义报错
- `#include "xxx.h"`(双引号)编译器优先在==当前工程目录==查找
- `#include <xxx.h>`(尖括号)编译器优先去==系统库路径==查找
## .c与.h文件
### .c(源文件)->实现
- 定义函数(函数本体)
- 定义全局变量/数组(真正分配内存)
- 不能重复定义！
### .h(头文件)->接口
- 声明函数
- 用extern声明数组/变量
- 放#define、结构体、枚举
- 作用：让其他文件能使用里面的东西
## 跨文件使用规则
### 函数
```c
//.c
void LED_Init(void)
{
	//代码
}

//.h
void LED_Init(void);

//其他文件
#include 'LED.h'
```
### 变量/数组
```c
//.c
uint8_t Serial_TxPacket[4];

//.h
extern uint8_t Serial_TxPacket[];
```
## 头文件保护宏(每个.h文件必写)
```c
#ifndef __LED_H
#define __LED_H
//声明内容(按照上面写的跨文件使用规则写)
#endif
```
作用：防止同一个.h文件被包含多次报错
