|概念|本质|举例（`lst = [10, 20]`）|
|---|---|---|
|列表地址（容器地址）|列表对象本身的内存地址（`id(lst)`），列表是「存储元素地址的容器」|`id(lst)=0x7f8a12345900`（容器地址）|
|元素地址（内容地址）|列表中每个元素对象的内存地址（`id(lst[i])`），元素是独立于列表的对象|`id(lst[0])=0x9789920`（10 的地址）|

| Python 概念          | C 语言等价概念              | 通俗解释            |
| ------------------ | --------------------- | --------------- |
| 列表地址（`id(lst)`）    | 指针数组的首地址（`arr`）       | 容器本身的地址         |
| 元素地址（`id(lst[0])`） | 指针数组中单个指针的值（`arr[0]`） | 容器里存的 “内容地址”    |
| 变量`lst`            | 指向数组首地址的指针（`p`）       | 只存容器地址，不存内容     |
| `lst[0]`           | `*arr[0]`（解引用）        | 通过容器地址→元素地址→拿到值 |
- Python 中「列表地址」是容器地址，「元素地址」是内容地址，二者层级不同：变量指向列表地址，列表地址内部存储元素地址；
- Python 的`a=b`：若`b`是整数，`a`/`b`指向同一元素地址；若`b`是列表，`a`/`b`指向同一列表地址（容器地址）；
- C 语言无 “列表”，但 Python 列表≈C 指针数组：Python 列表地址≈C 指针数组首地址，Python 元素地址≈C 指针数组中指针存储的地址；
- 核心差异：Python 自动处理地址解引用（写`lst[0]`就拿到值），C 需要手动用`*`解引用（`*arr[0]`）。
